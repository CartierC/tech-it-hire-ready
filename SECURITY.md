# Security Policy

## Scope

This repository is a public portfolio hub and documentation index for technical projects related to cloud, infrastructure, security, and automation.

## Reporting a Vulnerability

If you discover a security issue related to scripts, workflow files, or documentation in this repository, please do not open a public issue describing exploit details.

Instead, report the issue privately by contacting the repository owner through GitHub.

## Supported Content

This repository is maintained as a portfolio project. Security reviews are best-effort and focus on:

- shell scripts in `scripts/`
- GitHub Actions workflows in `.github/workflows/`
- documentation that may reference infrastructure or validation procedures

## Expectations

- Do not commit secrets, credentials, tokens, or private keys
- Use sample values or placeholders in public examples
- Review workflow changes before merging
- Prefer least-privilege defaults in all future automation examples

## Response Process

Reported issues will be reviewed, validated, and remediated as appropriate. Public disclosure should wait until the issue has been assessed and addressed.
